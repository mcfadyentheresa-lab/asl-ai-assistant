Implement task #52 only: Visible Navigation + Adaptive Layout

Context:
This is a live Aster & Spruce Connect app with existing populated data and working role-based behavior.
This task is a UI and layout improvement only.
Do not treat this as a rewrite.
Make safe, incremental, non-destructive changes.

Primary goal:
Fix the app’s wayfinding and device usability by adding visible role-aware navigation and adaptive desktop/tablet/phone layouts.

Core problems to solve:
1. Primary navigation is hidden inside the avatar dropdown, which makes the app harder to learn and navigate.
2. The app currently behaves too much like a squished desktop on phone.
3. The Planning Board is not helpful on phone and should not try to support full editing there.

Success criteria:
- Desktop has a persistent left sidebar with grouped, role-aware navigation.
- Tablet has a compact or collapsible version of the sidebar.
- Phone has a clear mobile drawer or bottom navigation for core destinations.
- Navbar becomes account-focused instead of carrying the primary app navigation.
- Dashboard has polished role-specific empty states.
- ProjectDetails is more usable on phone through layout adjustments only.
- Planning Board is view-only or limited on phone, with a clear message that full editing is for desktop/tablet.
- Dense tools show a “best experienced on desktop” banner on phone instead of feeling broken.
- All current routes, role guards, auth flows, and data remain intact.

Critical rules:
- Preserve all existing data.

- Do not change database schema or API contracts for this task.
- Do not refactor business logic in a risky way.
- Do not remove existing routes.
- Do not weaken access control or role guards.
- Do not change branding, colors, typography, or tone unless required for layout clarity.
